01

O que é o Keycloak?

O Keycloak é uma solução open-source de Identity and Access Management (IAM). Centraliza autenticação, autorização e gestão de utilizadores, permitindo que aplicações diferentes (Laravel, CodeIgniter, NestJS, etc.) partilhem o mesmo sistema de login sem reimplementar lógica de segurança em cada projeto.

Autenticação JWT

Emissão de access_token e refresh_token seguindo o padrão OpenID Connect / OAuth 2.0.

Gestão Centralizada

Utilizadores, grupos, roles e permissões geridos num único lugar para todos os sistemas.

Refresh de Tokens

Renovação automática de tokens sem necessidade de novo login do utilizador.

Introspection

Verificação de validade de tokens em tempo real nos middlewares das aplicações.

03

Endpoints OpenID Connect

Todos os endpoints seguem o padrão OpenID Connect. A URL base é:

url
http://localhost:8080/realms/{REALM}/protocol/openid-connect
Substitua {REALM} pelo nome do seu realm, ex: minha-app. O nome é case-sensitive.

Login (Autenticação)

POST /realms/{realm}/protocol/openid-connect/token

Autentica um utilizador com email e password e retorna os tokens de acesso.

Parâmetros do Request

Campo Valor
Content-Type application/x-www-form-urlencoded
grant_type password (fixo)
client_id ID do client configurado no Keycloak
client_secret Secret do client (aba Credentials)
username Email ou username do utilizador
password Password do utilizador

Exemplo cURL

bash 
curl -X POST http://localhost:8080/realms/minha-app/protocol/openid-connect/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=password" \
  -d "client_id=minha-app-client" \
  -d "client_secret=SEU_SECRET" \
  -d "username=user@email.com" \
  -d "password=senha123"

Resposta 200 OK

json
{
  "access_token":      "eyJhbGciOiJSUzI1NiIsInR...",
  "expires_in":         300,
  "refresh_expires_in": 1800,
  "refresh_token":     "eyJhbGciOiJIUzUxMiIsInR...",
  "token_type":        "Bearer",
  "session_state":     "abc-123",
  "scope":            "profile email"
}

Possíveis Erros

Status Descrição
401 Unauthorized Credenciais inválidas (password incorreta ou utilizador não existe)
400 Bad Request Parâmetros em falta ou client_id errado
401 invalid_client Client secret incorreto

Refresh Token

POST /realms/{realm}/protocol/openid-connect/token

Renova o access_token usando o refresh_token sem necessitar de nova password. Use quando o access_token expirar (por padrão a cada 5 minutos).

bash 
curl -X POST http://localhost:8080/realms/minha-app/protocol/openid-connect/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=refresh_token" \
  -d "client_id=minha-app-client" \
  -d "client_secret=SEU_SECRET" \
  -d "refresh_token=eyJhbGciOiJIUzUxMiIsInR..."
Retorna os mesmos campos do login com um novo access_token e refresh_token renovados.

Verificar Token (Introspection)

POST /realms/{realm}/protocol/openid-connect/token/introspect

Verifica se um token é válido e retorna informações sobre ele. Usado em middlewares para proteger rotas.

bash 
curl -X POST http://localhost:8080/realms/minha-app/protocol/openid-connect/token/introspect \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "token=eyJhbGciOiJSUzI1NiIsInR..." \
  -d "client_id=minha-app-client" \
  -d "client_secret=SEU_SECRET"

Token Válido

json
{
  "active":               true,
  "sub":                  "uuid-do-utilizador",
  "email":               "user@email.com",
  "preferred_username":  "user@email.com",
  "realm_access": { "roles": ["USER", "offline_access"] },
  "exp":                  1712345678,
  "iat":                  1712345378
}

Token Inválido / Expirado

json
{ "active": false }

Logout

POST /realms/{realm}/protocol/openid-connect/logout

Invalida a sessão do utilizador e revoga o refresh_token no servidor Keycloak.

bash 
curl -X POST http://localhost:8080/realms/minha-app/protocol/openid-connect/logout \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "client_id=minha-app-client" \
  -d "client_secret=SEU_SECRET" \
  -d "refresh_token=eyJhbGciOiJIUzUxMiIsInR..."
Resposta 204 No Content em caso de sucesso. Após o logout, o refresh_token não pode mais ser utilizado.
04

Admin API — Gestão de Utilizadores

A Admin API permite gerir utilizadores, roles e grupos programaticamente. A URL base é diferente:

url
http://localhost:8080/admin/realms/{realm}
Em produção, nunca exponha a Admin API diretamente ao frontend. Use sempre através do seu backend.

Criar Utilizador

POST /admin/realms/{realm}/users
bash 
curl -X POST http://localhost:8080/admin/realms/minha-app/users \
  -H "Authorization: Bearer {admin_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "username":      "joao.silva",
    "email":         "joao@empresa.com",
    "firstName":     "Joao",
    "lastName":      "Silva",
    "enabled":       true,
    "emailVerified": true,
    "credentials": [{
      "type":      "password",
      "value":     "senha123",
      "temporary": false
    }],
    "attributes": {
      "position": ["Desenvolvedor"]
    }
  }'

Listar / Buscar Utilizadores

GET /admin/realms/{realm}/users
bash 
# Listar todos
curl -X GET http://localhost:8080/admin/realms/minha-app/users \
  -H "Authorization: Bearer {admin_token}"

# Filtrar por email (exact match)
curl -X GET "http://localhost:8080/admin/realms/minha-app/users?email=joao@empresa.com&exact=true" \
  -H "Authorization: Bearer {admin_token}"

# Buscar por ID
curl -X GET http://localhost:8080/admin/realms/minha-app/users/{userId} \
  -H "Authorization: Bearer {admin_token}"

Atualizar Utilizador

PUT /admin/realms/{realm}/users/{userId}
bash 
curl -X PUT http://localhost:8080/admin/realms/minha-app/users/{userId} \
  -H "Authorization: Bearer {admin_token}" \
  -H "Content-Type: application/json" \
  -d '{ "firstName": "Joao", "lastName": "Atualizado" }'

Alterar Password

PUT /admin/realms/{realm}/users/{userId}/reset-password
bash 
curl -X PUT http://localhost:8080/admin/realms/minha-app/users/{userId}/reset-password \
  -H "Authorization: Bearer {admin_token}" \
  -H "Content-Type: application/json" \
  -d '{ "type": "password", "value": "novaSenha123", "temporary": false }'

Atribuir Role a Utilizador

POST /admin/realms/{realm}/users/{userId}/role-mappings/realm
bash 
# 1. Listar roles para obter o ID
curl -X GET http://localhost:8080/admin/realms/minha-app/roles \
  -H "Authorization: Bearer {admin_token}"

# 2. Atribuir role
curl -X POST http://localhost:8080/admin/realms/minha-app/users/{userId}/role-mappings/realm \
  -H "Authorization: Bearer {admin_token}" \
  -H "Content-Type: application/json" \
  -d '[{ "id": "uuid-da-role", "name": "ADMIN" }]'

Serviço Keycloak

Criar app/Services/KeycloakService.php:

php · app/Services/KeycloakService.php 
<?php
namespace App\Services;

use Illuminate\Support\Facades\Http;

class KeycloakService
{
    private string $tokenUrl;
    private string $baseUrl;
    private string $realm;
    private string $clientId;
    private string $clientSecret;

    public function __construct()
    {
        $this->baseUrl       = config('keycloak.url');
        $this->realm         = config('keycloak.realm');
        $this->clientId      = config('keycloak.client_id');
        $this->clientSecret  = config('keycloak.client_secret');
        $this->tokenUrl      = "{$this->baseUrl}/realms/{$this->realm}/protocol/openid-connect/token";
    }

    public function login(string $email, string $password): array
    {
        $response = Http::asForm()->post($this->tokenUrl, [
            'grant_type'    => 'password',
            'client_id'     => $this->clientId,
            'client_secret' => $this->clientSecret,
            'username'      => $email,
            'password'      => $password,
        ]);

        if ($response->failed()) {
            throw new \Exception('Login failed: ' . $response->body());
        }

        return $response->json();
    }

    public function refreshToken(string $refreshToken): array
    {
        $response = Http::asForm()->post($this->tokenUrl, [
            'grant_type'    => 'refresh_token',
            'client_id'     => $this->clientId,
            'client_secret' => $this->clientSecret,
            'refresh_token' => $refreshToken,
        ]);

        if ($response->failed()) {
            throw new \Exception('Refresh failed: ' . $response->body());
        }

        return $response->json();
    }

    public function verifyToken(string $token): array
    {
        $url = "{$this->baseUrl}/realms/{$this->realm}/protocol/openid-connect/token/introspect";

        $response = Http::asForm()->post($url, [
            'token'         => $token,
            'client_id'     => $this->clientId,
            'client_secret' => $this->clientSecret,
        ]);

        return $response->json();
    }

    public function logout(string $refreshToken): void
    {
        $url = "{$this->baseUrl}/realms/{$this->realm}/protocol/openid-connect/logout";

        Http::asForm()->post($url, [
            'client_id'     => $this->clientId,
            'client_secret' => $this->clientSecret,
            'refresh_token' => $refreshToken,
        ]);
    }
}

Middleware de Autenticação

Criar app/Http/Middleware/KeycloakAuth.php:

php · KeycloakAuth.php 
<?php
namespace App\Http\Middleware;

use Closure;
use App\Services\KeycloakService;

class KeycloakAuth
{
    public function handle($request, Closure $next)
    {
        $token = $request->bearerToken();

        if (!$token) {
            return response()->json(['message' => 'Token não fornecido'], 401);
        }

        $keycloak   = new KeycloakService();
        $introspect = $keycloak->verifyToken($token);

        if (empty($introspect['active'])) {
            return response()->json(['message' => 'Token inválido ou expirado'], 401);
        }

        $request->merge(['keycloak_user' => $introspect]);

        return $next($request);
    }
}

Controlador e Rotas

php · AuthController.php 
<?php
namespace App\Http\Controllers;

use App\Services\KeycloakService;
use Illuminate\Http\Request;

class AuthController extends Controller
{
    public function login(Request $request)
    {
        $request->validate([
            'email'    => 'required|email',
            'password' => 'required',
        ]);

        try {
            $tokens = (new KeycloakService())->login($request->email, $request->password);
            return response()->json($tokens);
        } catch (\Exception $e) {
            return response()->json(['message' => 'Credenciais inválidas'], 401);
        }
    }

    public function refresh(Request $request)
    {
        $request->validate(['refresh_token' => 'required']);
        try {
            $tokens = (new KeycloakService())->refreshToken($request->refresh_token);
            return response()->json($tokens);
        } catch (\Exception $e) {
            return response()->json(['message' => 'Refresh inválido'], 401);
        }
    }

    public function logout(Request $request)
    {
        $request->validate(['refresh_token' => 'required']);
        (new KeycloakService())->logout($request->refresh_token);
        return response()->json(['message' => 'Logout realizado']);
    }
}

Criar app/Services/KeycloakService.php:

php · app/Services/KeycloakService.php 
<?php
namespace App\Services;

use GuzzleHttp\Client;

class KeycloakService
{
    private Client $http;
    private string $tokenUrl;
    private string $baseUrl;
    private string $realm;
    private string $clientId;
    private string $clientSecret;

    public function __construct()
    {
        $this->http         = new Client(['verify' => false]);
        $this->baseUrl      = env('keycloak.url');
        $this->realm        = env('keycloak.realm');
        $this->clientId     = env('keycloak.clientId');
        $this->clientSecret = env('keycloak.clientSecret');
        $this->tokenUrl     = "{$this->baseUrl}/realms/{$this->realm}/protocol/openid-connect/token";
    }

    public function login(string $email, string $password): array
    {
        $resp = $this->http->post($this->tokenUrl, [
            'form_params' => [
                'grant_type'    => 'password',
                'client_id'     => $this->clientId,
                'client_secret' => $this->clientSecret,
                'username'      => $email,
                'password'      => $password,
            ]
        ]);
        return json_decode($resp->getBody(), true);
    }

    public function refreshToken(string $refreshToken): array
    {
        $resp = $this->http->post($this->tokenUrl, [
            'form_params' => [
                'grant_type'    => 'refresh_token',
                'client_id'     => $this->clientId,
                'client_secret' => $this->clientSecret,
                'refresh_token' => $refreshToken,
            ]
        ]);
        return json_decode($resp->getBody(), true);
    }

    public function verifyToken(string $token): array
    {
        $url  = "{$this->baseUrl}/realms/{$this->realm}/protocol/openid-connect/token/introspect";
        $resp = $this->http->post($url, [
            'form_params' => [
                'token'         => $token,
                'client_id'     => $this->clientId,
                'client_secret' => $this->clientSecret,
            ]
        ]);
        return json_decode($resp->getBody(), true);
    }

    public function logout(string $refreshToken): void
    {
        $url = "{$this->baseUrl}/realms/{$this->realm}/protocol/openid-connect/logout";
        $this->http->post($url, [
            'form_params' => [
                'client_id'     => $this->clientId,
                'client_secret' => $this->clientSecret,
                'refresh_token' => $refreshToken,
            ]
        ]);
    }
}

Filtro de Autenticação

Criar app/Filters/KeycloakFilter.php:

php · app/Filters/KeycloakFilter.php 
<?php
namespace App\Filters;

use App\Services\KeycloakService;
use CodeIgniter\Filters\FilterInterface;

class KeycloakFilter implements FilterInterface
{
    public function before($request, $arguments = null)
    {
        $token = $request->getHeaderLine('Authorization');
        $token = str_replace('Bearer ', '', $token);

        if (!$token) {
            return service('response')
                ->setStatusCode(401)
                ->setJSON(['message' => 'Token não fornecido']);
        }

        $data = (new KeycloakService())->verifyToken($token);

        if (empty($data['active'])) {
            return service('response')
                ->setStatusCode(401)
                ->setJSON(['message' => 'Token inválido ou expirado']);
        }

        session()->set('keycloak_user', $data);
    }

    public function after($request, $response, $arguments = null) {}
}

Fluxo de Autenticação Recomendado

1
Login
POST /token com grant_type=password → guardar access_token (5 min) e refresh_token (30 min) no cliente.
2
Requisição Autenticada
Enviar o access_token em cada request: Authorization: Bearer {token}
3
Middleware Valida
Backend chama /introspect e verifica se active === true. Se falso → 401.
4
Token Expirado
Se o backend retornar 401, o cliente usa o refresh_token para obter um novo access_token automaticamente.
5
Refresh Expirado
Se o refresh também falhar, forçar novo login do utilizador.
6
Logout
POST /logout com o refresh_token para revogar a sessão no servidor.

Resolução de Problemas

Problema Solução
401 no login Verificar username/password. Confirmar que o utilizador está enabled no Keycloak.
invalid_client Client secret incorrecto. Ir a Clients → Credentials → regenerar secret.
Token expirado (active: false) Usar o refresh_token para obter novo access_token. Se falhar, fazer novo login.
CORS error no browser Configurar Web Origins: Clients → seu-client → Access Settings → Web Origins: *
Connection refused Verificar se o Keycloak está a correr: docker ps ou netstat -tlnp | grep 8080
Realm não encontrado O nome do realm é case-sensitive. Confirmar na interface admin.
Admin API 403 Forbidden Token admin expirado. Autenticar novamente no realm master.
Grant type not allowed Activar "Direct Access Grants" no client: Clients → seu-client → Settings.
